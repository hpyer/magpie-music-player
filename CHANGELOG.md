# 变动历史

本文件记录面向用户和插件开发者的重要变更。每次发布新版本前，应根据 git 历史整理对应版本的变动内容。

格式约定：

- 桌面应用版本使用 `## v1.2.0 - YYYY-MM-DD`
- npm 包版本也记录在本文件中，并在标题中标明包名
- GitHub Release 只会读取桌面应用 tag 对应的应用版本段落，例如 `v1.2.0` 读取 `## v1.2.0 - ...`

## v1.1.0 - 2026-07-23

### 新增

- 新增“我的收藏”虚拟播放列表，并按歌曲来源同步本地与远程收藏状态。
- Navidrome 官方插件支持拉取远程收藏状态，并在收藏或取消收藏远程歌曲时同步到 Navidrome 服务器。
- 随机播放支持为收藏歌曲设置更高播放权重，可在设置中选择权重比例。
- 设置页新增启动恢复和切歌淡入淡出选项，支持恢复上次歌曲或继续播放，并可调整淡入淡出效果。
- 远程播放列表编辑弹窗支持立即刷新歌曲，便于更新插件播放列表内容。

### 优化

- 播放列表下拉箭头和下拉菜单增加动画效果。
- 优化播放列表工具按钮悬停时的图标居中效果。
- 将收藏歌曲随机权重改为合并式单选按钮组。
- 扩展 README 安装说明。

## @magpie-music/plugin-types v1.1.0 - 2026-07-23

### 新增

- 新增 `media:favorite` 权限、`media-favorite` 插件能力、`media.favorite` 消息类型和 `setMediaFavorite` 插件方法，用于同步远程媒体收藏状态。

## v1.0.2 - 2026-07-22

### 新增

- Navidrome 官方插件支持按远程歌曲读取封面和歌词，并发布 `1.0.1` 插件包。
- 支持远程媒体边播放边缓存，同一首未变化歌曲只缓存一次，并在播放进度条展示缓存进度。

### 修复

- 修复禁用播放列表插件后，对应远程播放列表仍可在下拉列表中选择的问题。
- 修复远程媒体缓存完成后进度条颜色叠加变深的问题，并优化深色主题下的缓存进度显示。
- 修复启动时自动为所有播放列表加载封面和歌词缓存的问题，改为在播放列表或歌词面板展示时按需加载。

### 优化

- 统一缓存文件后缀：歌曲使用 `.song`，封面使用 `.cover`，歌词使用 `.lyric`。
- 更新官方插件下载链接和截图展示尺寸。

## @magpie-music/plugin-types v1.0.1 - 2026-07-22

### 新增

- 新增 `media-lyrics`、`media-cover` 插件能力和 `media.lyric`、`media.cover` 消息类型，用于按媒体读取歌词和封面。

## v1.0.1 - 2026-07-21

### 修复

- 修复播放列表下拉框箭头未固定在右侧的问题。
- 修复播放列表弹窗中选择远程播放列表后下拉菜单未自动收起的问题。
- 取消播放列表和设置表单描述文字的控件触发行为，减少误触。
- 优化新增播放列表默认名称，并将播放列表类型移到名称上方。
- 修复桌面窗口位置跨启动持久化问题。

### 优化

- 延后插件启动同步，减少应用启动阶段的阻塞。
- 调整 Linux AppImage 构建流程，避免依赖 FUSE 运行 AppImage 工具。

## v1.0.0 - 2026-07-21

### 新增

- 初始发布 Magpie Music Player 桌面应用。
- 支持本地音乐目录扫描、播放列表、播放控制和基础设置。
- 支持插件安装、配置、启用、禁用和删除。
- 支持插件黑名单拉取、签名校验、本地缓存和阻止提示。
- 提供 `@magpie-music/plugin-types` 插件类型包。
- 提供 `@magpie-music/plugin-cli` 插件创建、测试和打包命令。
